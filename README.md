
● 项目总览

  这是一个 仿 QQ 的即时通讯 + 个人网盘系统，基于 Qt 5.14.2（C++）开发，Client-Server 架构。

  ---
  技术栈

  ┌─────────┬─────────────────────────────────────────────────────┐
  │  层面   │                        技术                         │
  ├─────────┼─────────────────────────────────────────────────────┤
  │ 语言    │ C++11                                               │
  ├─────────┼─────────────────────────────────────────────────────┤
  │ UI 框架 │ Qt 5.14.2（Widgets + Network + Sql）                │
  ├─────────┼─────────────────────────────────────────────────────┤
  │ 通信    │ TCP Socket，自定义二进制协议                        │
  ├─────────┼─────────────────────────────────────────────────────┤
  │ 数据库  │ MySQL（通过 QSqlDatabase + QMYSQL 驱动）            │
  ├─────────┼─────────────────────────────────────────────────────┤
  │ 并发    │ QThreadPool + QRunnable（服务端每个客户端一个线程） │
  ├─────────┼─────────────────────────────────────────────────────┤
  │ 构建    │ qmake（.pro 文件）                                  │
  ├─────────┼─────────────────────────────────────────────────────┤

  ---
  自定义通信协议 (protocol.h)

  两端共用同一份协议定义，核心是 PDU（协议数据单元）：

  ┌──────────────┬──────────────┬──────────────┬──────────────┬──────────┐
  │  uiPDULen    │  uiMsgLen    │  uiMsgType   │   caData[64] │ caMsg[]  │
  │  (总长度)     │  (消息长度)   │  (消息类型)   │   (参数区)    │ (柔性数组) │
  │  4 bytes     │  4 bytes     │  4 bytes     │   64 bytes   │ 变长      │
  └──────────────┴──────────────┴──────────────┴──────────────┴──────────┘

  消息类型枚举共 32 种，分为 Request/Respond 对：

  ┌─────────────────┬──────────────────────────┬────────────────────────┐
  │    业务模块     │       Request 类型       │          用途          │
  ├─────────────────┼──────────────────────────┼────────────────────────┤
  │ 注册            │ REGIST_REQUEST           │ 客户端发送用户名+密码  │
  ├─────────────────┼──────────────────────────┼────────────────────────┤
  │ 登录            │ LOGIN_REQUEST            │ 客户端发送认证信息     │
  ├─────────────────┼──────────────────────────┼────────────────────────┤
  │ 查找用户        │ FIND_USER_REQUEST        │ 查询用户在线状态       │
  ├─────────────────┼──────────────────────────┼────────────────────────┤
  │ 在线用户        │ ONLINE_USER_REQUEST      │ 获取所有在线用户列表   │
  ├─────────────────┼──────────────────────────┼────────────────────────┤
  │ 添加好友        │ ADD_FRIEND_REQUEST       │ 发送好友申请           │
  ├─────────────────┼──────────────────────────┼────────────────────────┤
  │ 同意好友        │ ADD_FRIEND_ARGEE_REQUEST │ 同意好友申请           │
  ├─────────────────┼──────────────────────────┼────────────────────────┤
  │ 刷新好友        │ FLUSH_FRIEND_REQUEST     │ 刷新好友列表           │
  ├─────────────────┼──────────────────────────┼────────────────────────┤
  │ 删除好友        │ DEL_FRIEND_REQUEST       │ 删除好友关系           │
  ├─────────────────┼──────────────────────────┼────────────────────────┤
  │ 聊天            │ CHAT_REQUEST             │ 发送聊天消息           │
  ├─────────────────┼──────────────────────────┼────────────────────────┤
  │ 新建文件夹      │ MKDIR_REQUEST            │ 在网盘创建文件夹       │
  ├─────────────────┼──────────────────────────┼────────────────────────┤
  │ 刷新文件        │ FLUSH_FILE_REQUEST       │ 刷新当前目录文件列表   │
  ├─────────────────┼──────────────────────────┼────────────────────────┤
  │ 删除目录        │ DEL_DIR_REQUEST          │ 删除网盘文件夹         │
  ├─────────────────┼──────────────────────────┼────────────────────────┤
  │ 上传文件-初始化 │ UPLOAD_FILE_INIT_REQUEST │ 上传前告知文件名和大小 │
  ├─────────────────┼──────────────────────────┼────────────────────────┤
  │ 上传文件-数据   │ UPLOAD_FILE_DATA_REQUEST │ 传输文件数据块         │
  ├─────────────────┼──────────────────────────┼────────────────────────┤
  │ 分享文件        │ SHARE_FILE_REQUEST       │ 分享文件给好友         │
  ├─────────────────┼──────────────────────────┼────────────────────────┤
  │ 接收文件        │ SHARE_FILE_ARGEE_REQUEST │ 同意接收分享文件       │
  └─────────────────┴──────────────────────────┴────────────────────────┘

  caData[64] 前 32 字节通常放当前用户名，后 32 字节放目标用户名或布尔/整型结果。caMsg
  放变长数据（文件路径、好友列表、聊天内容等）。

  ---
  关键设计细节

  1. 单例模式

  Client、Server、Index、OperateDb、MyQTcpServer 全部使用单例，禁止拷贝和赋值。

  2. 粘包处理

  Client（client.cpp:248-256）和 Server（mytcpsocket.cpp:146-156）都使用 QByteArray buffer 缓冲区处理 TCP 粘包：先读完整
   PDU 头获取总长度，不够则等待下次数据到达再拼接。

  3. 服务端并发模型

  MyQTcpServer 持有 QThreadPool（最大 8 线程），每个客户端连接进来时创建 MyTcpSocket + ClientTask（QRunnable），将
  socket 移到子线程处理，避免阻塞主线程。

  4. 消息转发

  聊天和好友申请等场景，服务器收到消息后通过 resend() 遍历 m_socketList，根据目标用户名匹配对应的 socket 转发。

  5. 文件上传流程

  分两步：先发 INIT 请求（告知文件名+大小）让服务器创建文件，再循环发送 DATA 块（每块 4096
  字节），服务器逐块写入，接收完毕后关闭文件。客户端使用 Uploader 类将上传放入独立线程，通过信号槽通知主线程。

  6. 文件分享

  分享者选择文件和好友 → 服务器遍历目标好友列表逐一转发分享请求 → 接收者同意后服务器执行 QFile::copy()
  将文件复制到接收者目录。

  ---
  配置

  两端都通过 .config 文件配置（嵌入 Qt 资源文件）：

  Client（client.config）：服务器 IP、端口、本地文件根路径

  Server（server.config）：监听 IP、端口、用户文件存储根路径、数据库连接参数

  ---
  数据库表

  - user_info — id、name、pwd、online（0 离线 / 1 在线）
  - friend — id、user_id、friend_id（双向好友关系通过两条记录的 union 查询实现）
